# Release Check Skill

Verify:
1. Scope matches the intended milestone
2. Changed files are explained
3. Manual or automated checks are listed honestly
4. Docs are updated if behavior changed
5. Licensing / third-party notes are updated if dependencies changed
6. Known risks are called out

Output:
- Ready / not ready
- What was checked
- What is still missing
- Recommended next step
