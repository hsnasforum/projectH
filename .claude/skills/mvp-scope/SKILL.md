# MVP Scope Skill

When invoked, produce:
1. One-line product definition
2. Target user
3. Core problems solved
4. MVP in-scope features
5. Explicit non-goals
6. Risks
7. Milestone split
8. Acceptance criteria

Rules:
- Cut aggressively
- Prefer 1-person build scope
- Separate product value from technical ambition
- Flag anything that depends on cloud infrastructure, destructive automation, or unclear permissions
