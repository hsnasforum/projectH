이 프로젝트의 목표는 로컬 우선 AI 비서 MVP를 만드는 것입니다. 제품은 사용자의 PC에서 동작하며, 문서를 읽고 요약하고 검색하고, 승인 기반으로 제한된 작업만 수행해야 합니다.

항상 다음 원칙을 따르세요.

1. 범위를 함부로 넓히지 않습니다.
- 먼저 현재 요청이 MVP 범위인지 판단합니다.
- 불필요한 기능 추가, 대규모 리팩터링, 추상화 과잉을 피합니다.
- 작은 변경으로 해결 가능하면 작은 변경을 우선합니다.

2. 로컬 우선 원칙을 유지합니다.
- 클라우드 의존 기능은 명시적 요청이 있을 때만 제안합니다.
- 모델, 런타임, 저장소는 교체 가능한 구조로 유지합니다.
- 특정 공급자나 특정 모델에 강하게 결합하지 않습니다.

3. 안전성을 우선합니다.
- 기본 동작은 읽기 중심이어야 합니다.
- 파일 수정, 이동, 삭제, 셸 실행, 네트워크 사용은 위험 작업입니다.
- 위험 작업은 승인 없이 자동화하지 않습니다.
- 파괴적 변경은 항상 사전 설명과 영향 범위를 먼저 제시합니다.

4. 상업화와 권리 구조를 고려합니다.
- 코드 라이선스, 모델 라이선스, 자산 라이선스, 상표를 분리해서 봅니다.
- 특정 오픈소스 이름이나 브랜드를 제품 정체성으로 전제하지 않습니다.
- 자체 브랜드, 자체 UX, 자체 워크플로를 우선합니다.

5. 응답 방식
- 사용자에게는 항상 한국어 존댓말로 답합니다.
- 사실, 가정, 권고를 구분합니다.
- 확실하지 않으면 불확실성을 명시하고 가정을 적습니다.
- 구현 전에는 목표, 변경 범위, 예상 영향, 검증 방법을 먼저 정리합니다.
- 구현 후에는 변경 파일, 변경 이유, 리스크, 검증 결과를 요약합니다.

6. 기술 원칙
- UI, 에이전트 로직, 도구 레이어, 저장소 레이어를 분리합니다.
- 모델 호출부는 추상화해서 교체 가능하게 만듭니다.
- 승인 기반 실행 흐름과 작업 로그를 핵심 기능으로 유지합니다.
- 삭제, 덮어쓰기, 자율 실행형 에이전트는 기본값으로 넣지 않습니다.
